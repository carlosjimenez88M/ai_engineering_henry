# 03 - Ejercicios tipo LeetCode

Objetivo: entrenar pensamiento algoritmico bajo presion, con explicaciones que justifiquen cada decision.

## Critica del estado inicial (y mejora aplicada)

Problema detectado:
- Habia buena cobertura de problemas, pero faltaba un criterio de evaluacion para simulacion de entrevista.

Mejora aplicada:
- Se explicita un rubric de calidad para cada solucion.

## Rubric de evaluacion (para cada problema)

1. Correctitud:
- Resuelve casos normales y casos borde.

2. Complejidad:
- Puedes justificar tiempo y espacio sin improvisar.

3. Claridad:
- Variables y flujo comprensibles en voz alta.

4. Robustez:
- Maneja entradas invalidas cuando aplica.

## Archivos

- `ejercicios.md` - explicaciones detalladas y alternativas.
- `leetcode_runnable.py` - implementaciones ejecutables.
- `test_leetcode.py` - pruebas automatizadas.

## Aplicacion AI/ML

Aunque no son problemas de ML directos, los patrones aqui aparecen en:

- Preprocesamiento de datos.
- Ventanas deslizantes para streaming.
- Hashing para deduplicacion.
- Busqueda eficiente en indices y caches.
