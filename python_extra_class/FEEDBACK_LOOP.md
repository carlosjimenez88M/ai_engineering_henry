# Feedback Loop

Objetivo: iterar hasta que el material quede perfecto para tu estilo.

Checklist rápida:
- Orden: ¿la secuencia te parece clara?
- Profundidad: ¿está muy básico o muy avanzado?
- Ejemplos: ¿faltan ejemplos reales de tu área?
- Ritmo: ¿quieres más ejercicios y menos teoría (o al revés)?
- Lenguaje: ¿prefieres más técnico o más simple?

Cómo trabajamos:
1. Dime qué parte quieres ajustar (por ejemplo: "agrega más ejemplos en funciones").
2. Yo propongo un cambio puntual.
3. Tú confirmas o ajustas.
4. Repetimos hasta cerrar.

Prompts útiles (elige uno y yo itero sobre eso):
- "Reescribe la sección de OOP para que sea más Pythonic (properties, type hints)."
- "En LeetCode, agrega 2 casos límite y una traza manual por ejercicio."
- "Haz las explicaciones más cortas pero más densas (menos texto, más idea)."
